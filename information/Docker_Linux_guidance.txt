A️⃣ Canonical command sequence

From Docker daemon start → app running

This is the official, recruiter-grade workflow for your project.

1️⃣ Start Docker Engine (daemon)
sudo systemctl start docker


(Optional but recommended at boot)

sudo systemctl enable docker


Verify daemon + permissions:

docker ps


✅ You should see an empty table, no errors.

2️⃣ Go to the project root
cd ~/work/Projects/nka-academic-management-system

3️⃣ Bring up the containers (first time or after changes)
docker compose up -d --build


This will start:

MariaDB

Redis

Laravel (PHP-FPM)

Nginx

Node/Vite

4️⃣ Install PHP dependencies (first run only)
docker compose exec laravel composer install

5️⃣ Generate Laravel app key (first run only)
docker compose exec laravel php artisan key:generate

6️⃣ Run migrations + seeders
docker compose exec laravel php artisan migrate --seed

7️⃣ Access the app

🌐 App: http://localhost:8000

🗄 phpMyAdmin: http://localhost:8080

⚡ Vite dev server: http://localhost:5173

8️⃣ Stop the app (safe, keeps volumes)
docker compose down


⚠️ Do NOT use -v unless you intentionally want to delete volumes.

9️⃣ Restart later (normal daily workflow)
sudo systemctl start docker
docker compose up -d
