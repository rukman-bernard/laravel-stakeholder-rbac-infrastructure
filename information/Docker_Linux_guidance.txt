Quick “proof” snippet you can paste into your documentation

Symptom: /storage/avatars/*.jpg returned 403/404.

Evidence: nginx logs showed failed (13: Permission denied) when accessing files under storage/app/public.

Cause: parent directory storage/app had mode 770, blocking traversal for nginx worker user.

Fix: change storage/app permissions to 775 so nginx can traverse and serve static assets.
